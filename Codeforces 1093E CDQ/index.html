Codeforces 1093E Intersection of Permutations

Overview

一方大哥又被打了……

题意

题目链接
给出两个$n$的排列$a,b$，以及$m$个操作：

  • $1$ $l_a$ $r_a$ $l_b$ $r_b$ : 求$|a_{l_a\dots r_a}\cap b_{l_b\dots r_b}|$
  • $2$ $x$ $y$: 交换$b_x, b_y$

CDQ分治

一股浓郁的lxl题面感，在线的做法也确实是毒瘤的树套树。但是，，，
我们可以离线操作啊！比如说LCA问题在线做法基本都是$O(nlogn)$的，但是离线做就可以$O(n)$.
这里就要涉及到一种神奇的思想——CDQ分治！我们知道一般的分治通常是这样的：

  • 把问题划分成$[L, M]$和$[M + 1, R]$
  • 区间长度是1就直接返回答案进行合并，否则继续拆分区间进行递归

而CDQ分治会把问题划分成下面几个部分：

  • $l\leq i \leq mid, l\leq j\leq mid$
  • $mid + 1\leq i\leq r, mid + 1\leq j\leq r$
  • $l\leq i \leq mid, mid + 1\leq j\leq r$

其中前两个问题可以递归解决，问题在于第三种情况。通常来说，CDQ分治利用左半边的信息求解右半边。下面来看一个经典问题：

给出序列$a$，求有多少$(i, j)$满足$i < j, a_i > a_j$

这是一个非常经典的逆序对问题，他还有一个名字叫做“二维偏序”，其中一种解决方法是归并，而归并的过程就是一个经典的CDQ分治：我们从右半区间取出一个数的时候，需要加上左半区间有多少数比它大。

三维偏序问题：给出$n$个三元组$(a, b, c)$，求所有的三元组$(x, y, z)$，满足$x < a, y < b, z < c$

和二维偏序问题很像，我们可以先按照$a$排序，然后对$b$进行归并，但是第三维的$c$却是个问题，我们并不能在对$b$的归并中得到$c$的信息。因此我们需要一个东西能快速维护“有多少三元组的$c$小于当前三元组”，同时还支持修改操作。和逆序对的另一种做法很像，我们选择好用的树状数组。注意每次归并结束都要清空树状数组，因为这一点我WA了好多次。
三维偏序问题可以扩展到二维数点问题，即：

平面上有$n$个点，问矩形$(x1, y1, x2, y2)$内有多少点

我们把每个点变成$(time, x, y)$的形式，其中$time$不妨就定成给出的顺序，然后就变成一个类似二维前缀和的东西查询。
